Py-Scout

A Windows-focused switchport lookup tool for LLDP/CDP discovery using tshark.

Py-Scout listens for LLDP and CDP advertisements, auto-selects a usable Ethernet adapter, and reports the first detected switch and port, plus the advertised neighbor IP address when available. It helps network engineers identify the switchport their laptop is connected to without manually building tshark commands.

What Py-Scout Does

Find the connected switchport faster.

Listens for discovery traffic

Captures LLDP and CDP advertisements with tshark and reads the neighbor details exposed by the connected network device.

Selects a practical adapter

Uses Windows adapter information and tshark interface listings to choose a usable Ethernet capture interface.

Reports the first match

Prints the detected protocol, switch, port, advertised neighbor IP address, status, and timeout details in human-readable or JSON output.

Requirements

Designed for Windows and tshark.

  • Windows
  • Python 3.10 or newer for source usage
  • Wireshark installed
  • tshark available on PATH
  • tkinter required for GUI mode
  • PyInstaller only required for building the executable

Install Wireshark with tshark support, then confirm tshark -v works from PowerShell before scanning.

CLI Usage

Run from source when Python is available.

Open PowerShell or Windows Terminal and pass command-line options to use CLI mode. Launching from source with no arguments opens the GUI. 

py .\py-scout.py --help
py .\py-scout.py --list
py .\py-scout.py --timeout 10
py .\py-scout.py --json --timeout 10
py .\py-scout.py --gui

Logging

Optional result logs for repeatable troubleshooting.

CSV and JSON output

Py-Scout can write logs to logs\py-scout-log.csv and logs\py-scout-log.json. CSV logging uses --log-csv; JSON logging uses --log-json.

Controlled location

--log-dir controls the output directory and defaults to logs. Failures during scan setup should still produce failure log entries with the shared schema.

Result records only

Logs record scan results, not raw packet captures. They are useful for tracking attempts without storing full capture files.

Safety and Privacy

Local execution, local output.

Py-Scout runs locally and does not upload captures or logs anywhere. Packet captures and network discovery data may contain sensitive environment details, so handle saved output carefully.

Limitations

Discovery depends on the local network path.

  • LLDP or CDP must be advertised for detection to work.
  • Some docks, pass-through phones, unmanaged switches, or filtered links may block discovery traffic.
  • Interface matching depends on Windows adapter names and tshark interface names lining up.
  • The tool is intentionally Windows-focused.
